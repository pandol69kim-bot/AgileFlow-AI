# Sub Agent System - 프로젝트 파이프라인

아이디어 하나를 입력하면, 9개의 전문 에이전트가 순차/병렬로 프로젝트 전체 공정을 수행합니다.

## 파이프라인 구조

```
아이디어 입력
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  00_orchestrator.md  │  총괄 오케스트레이터 (파이프라인 조율) │

    ┌──────────────────▼──────────────────┐

    │  아이디어 분석 & 정제                  │
    │  → 분석 보고서, 타겟 사용자, MVP 기능   │
    └──────────────────┬──────────────────┘
                       │
    ┌──────────────────▼──────────────────┐
    │  02_planner.md                      │
    │  기획서 작성                          │
    │  → PRD, User Story, 화면흐름, API 명세 │
    └──────────────────┬──────────────────┘
                       │
          ┌────────────┴────────────┐

┌─────────────────┐     ┌──────────────────────┐
│ 03_designer.md  │     │ 04_architect.md      │

│   와이어프레임    │     │   프로젝트 구조        │
└────────┬────────┘     └────────┬─────────────┘
         │                       │
         └───────────┬───────────┘
                     │
          ┌──────────┴──────────┐
          ▼                      ▼

    ┌──────────────────────────────────┐

    ┌──────────────────▼──────────────────┐

    └─────────────────────────────────────┘

                  최종 산출물
```

## 에이전트 목록

| # | 파일 | 역할 | 핵심 산출물 |
|---|------|------|-----------|
| 00 | `00_orchestrator.md` | 총괄 조율 | 파이프라인 실행 |
| 01 | `01_idea_analyst.md` | 아이디어 분석 | 분석 보고서 |
| 02 | `02_planner.md` | 기획 | PRD, User Story |
| 03 | `03_designer.md` | UI/UX 디자인 | 디자인 시스템, 와이어프레임 |
| 04 | `04_architect.md` | 시스템 설계 | 기술 스택, DB 스키마, 구조 |
| 05 | `05_frontend_dev.md` | 프론트엔드 개발 | 컴포넌트, 페이지, 훅 |
| 06 | `06_backend_dev.md` | 백엔드 개발 | API, 서비스, 스키마 |
| 07 | `07_tester.md` | 테스트/QA | 테스트 코드, 코드 리뷰 |
| 08 | `08_deployer.md` | 배포/인프라 | Docker, CI/CD, 모니터링 |
| 09 | `09_project_manager.md` | 프로젝트 관리 | 대시보드, 최종 리포트 |

## 사용 방법

### 1. 아이디어 입력
각 에이전트의 프롬프트 템플릿에 아이디어를 전달하여 순차 실행합니다.

### 2. 순차 실행
```
Step 1: 01_idea_analyst 에이전트에 아이디어 전달
Step 2: 01의 산출물을 02_planner에 전달
Step 3: 02의 산출물을 03_designer에, 01+02의 산출물을 04_architect에 병렬 전달
Step 4: 03+04의 산출물을 05_frontend_dev + 06_backend_dev에 병렬 전달
Step 5: 05+06의 산출물을 07_tester에 전달
Step 6: 모든 산출물을 08_deployer에 전달
Step 7: 전체 산출물을 09_project_manager에 전달
```

### 3. 산출물 확인
각 단계의 산출물은 `output/{agent_name}/` 디렉토리에 저장됩니다.

## 커스터마이징

- 각 에이전트 MD 파일의 프롬프트 템플릿을 수정하여 동작을 조정할 수 있습니다.
- 불필요한 에이전트는 파이프라인에서 제외할 수 있습니다.
- 프로젝트 성격에 따라 에이전트를 추가할 수 있습니다 (예: 마케팅, 법률 검토 등).
