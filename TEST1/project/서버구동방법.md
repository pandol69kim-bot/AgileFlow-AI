# AgileFlow 서버 구동 방법

## 전체 구조

```
project/
├── apps/
│   ├── web/       → 프론트엔드   Vite + React        :5173
│   ├── api/       → 백엔드 API   Node.js + Fastify   :3001
│   └── pipeline/  → AI 파이프라인 Python + FastAPI   :8000 (Docker)
├── scripts/
│   ├── start.sh   → 전체 시작 (Git Bash)
│   ├── start.bat  → 전체 시작 (Windows CMD)
│   └── stop.sh    → 전체 종료
└── docker-compose.yml  → postgres + redis + pipeline
```

## 빠른 시작 (권장)

### Git Bash

```bash
cd project
npm start
# 또는
bash scripts/start.sh
```

### Windows CMD

```
scripts\start.bat
```

스크립트가 아래 순서를 자동으로 처리합니다.

| 단계 | 내용 |
|------|------|
| 1 | `.env` 파일 없으면 `.env.example`에서 자동 복사 |
| 2 | Docker로 postgres + redis + pipeline 컨테이너 시작 |
| 3 | postgres / redis 헬스체크 통과 대기 |
| 4 | `prisma migrate deploy` 로 DB 스키마 적용 |
| 5 | API 서버 백그라운드 시작 (포트 3001) |
| 6 | 웹 서버 백그라운드 시작 (포트 5173) |

완료 후 접속 주소:

```
웹          http://localhost:5173
API         http://localhost:3001
API 문서    http://localhost:3001/docs
Pipeline    http://localhost:8000
```

**종료:** `Ctrl+C` — API·웹 프로세스 자동 정리 (Docker 컨테이너는 유지)

## 환경 변수 설정

`.env.example`을 복사해 `.env`를 만들고, 사용할 AI 공급자의 키를 입력합니다.

```bash
cp .env.example .env
```

```env
# Database
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/agileflow

# Redis
REDIS_URL=redis://localhost:6379

# Auth
JWT_SECRET=change-this-secret-in-production

# AI — 사용할 공급자 키만 입력하면 됩니다
ANTHROPIC_API_KEY=sk-ant-...      # Claude
OPENAI_API_KEY=sk-proj-...        # GPT-4.1
GOOGLE_API_KEY=AIza...            # Gemini 2.5

# LangSmith 트레이싱 (선택)
LANGCHAIN_API_KEY=ls__...
LANGCHAIN_TRACING_V2=true
LANGCHAIN_PROJECT=agileflow

# 서비스 URL
PIPELINE_URL=http://localhost:8000
WEB_ORIGIN=http://localhost:5173
VITE_API_URL=http://localhost:3001/api/v1
```

## 서비스별 개별 실행

자동 스크립트 없이 수동으로 각 서비스를 실행하는 방법입니다.

### 1. Docker 인프라 (postgres + redis + pipeline)

```bash
cd project
docker compose up -d
```

컨테이너 상태 확인:

```bash
docker compose ps
```

### 2. API 서버

```bash
cd project/apps/api
npm install          # 최초 1회
npx prisma migrate deploy   # DB 스키마 적용
npm run dev          # 개발 서버 시작 (:3001)
```

> **DB 초기 데이터 삽입이 필요한 경우:**
> ```bash
> npm run db:seed
> ```

### 3. 웹 서버

```bash
cd project/apps/web
npm install    # 최초 1회
npm run dev    # 개발 서버 시작 (:5173)
```

### 4. Pipeline (로컬 직접 실행 — 선택)

Docker 대신 로컬 Python 환경으로 실행하려면:

```bash
cd project/apps/pipeline

# 가상환경 생성 (최초 1회)
python -m venv venv
source venv/Scripts/activate   # Windows Git Bash
# 또는 venv\Scripts\activate   # Windows CMD

pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

## 완전 종료

```bash
# API + 웹 프로세스 + Docker 컨테이너 모두 종료
cd project
npm run stop
# 또는
bash scripts/stop.sh
```

Docker 컨테이너만 종료:

```bash
cd project
docker compose down
```

DB 볼륨까지 삭제 (초기화):

```bash
docker compose down -v
```

## 자주 발생하는 문제

### PostgreSQL 타임아웃

```
[ERR] PostgreSQL 시작 타임아웃
```

Docker Desktop이 실행 중인지 확인:

```bash
docker info
docker compose up -d postgres
docker compose logs postgres
```

### Prisma 마이그레이션 오류

```bash
cd project/apps/api
npx prisma migrate dev   # 개발 환경에서 재실행
```

### 포트 충돌

사용 중인 포트 확인 후 프로세스 종료:

```bash
# Windows
netstat -ano | findstr :3001
netstat -ano | findstr :5173
netstat -ano | findstr :8000

# Git Bash
lsof -i :3001
```
