# CLAUDE.md — AgileFlow 프로젝트 규칙

## 작업 로그 규칙

**매 작업 완료 후 반드시 로그를 작성한다.**

### 규칙

1. **작업내역 파일**: 신규 기능·구현 작업은 `작업내역_{프로젝트명}_{날짜}.md`에 기록
2. **수정작업 파일**: 버그 수정·오류 대응은 `수정작업.md`에 날짜별로 누적 기록
3. **형식**: 아래 템플릿을 따른다

### 수정작업 로그 템플릿 (`수정작업.md`)

```markdown
## YYYY-MM-DD

### {작업 제목}

**오류 내용** (오류 대응 시)
\```
{에러 메시지}
\```

**원인**
{원인 설명}

**수정 내용**
{수정 사항 설명}

**수정 파일**
| 파일 | 작업 |
|------|------|
| `경로/파일명` | 신규 생성 / 수정 / 삭제 |

**확인 방법**
\```bash
{검증 명령어}
\```
```

### 작업내역 로그 템플릿 (`작업내역_{프로젝트명}_{날짜}.md`)

```markdown
# 작업내역 — {프로젝트명}

- **날짜:** YYYY-MM-DD
- **작업 디렉토리:** `경로`

## 작업 요약
| 단계 | 작업 | 산출물 경로 |
|------|------|------------|

## {섹션별 상세 내용}

## 다음 작업 (기술 부채)
- [ ] 항목
```

## 프로젝트 구조

```
TEST1/
├── docs/             # 기획 문서 및 에이전트 산출물
├── project/          # 실제 구현 코드
│   ├── apps/web      # Vite + React
│   ├── apps/api      # Node.js + Fastify + Prisma
│   └── apps/pipeline # Python + LangGraph + FastAPI
├── 수정작업.md        # 버그/오류 수정 누적 로그
├── 작업내역_*.md      # 기능 구현 작업 로그
└── CLAUDE.md         # 이 파일
```

## 환경 변수

- 루트 `.env`: `project/.env`
- API 서비스 `.env`: `project/apps/api/.env` (Prisma 마이그레이션용)
- `.gitignore`에 모든 `.env` 파일 포함 확인 필수
